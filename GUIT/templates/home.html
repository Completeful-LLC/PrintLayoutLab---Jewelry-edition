How to get started with ShipStation and the AutoDesigner

  1. Access ShipStation using your personal login credentials.
  2. Choose the orders you wish to design, prioritizing them based on order date or priority level.
  3. Navigate to the "Other Actions" tab located at the top of the page, and find "Export Orders".
  4. Select the "Export Line Items" option, followed by "Item Skus W Options."
  5. Click on the "Export" button to complete the process.
  6. Proceed to the Precheck.
    7. Go to ShipStation

What to look for in Precheck

ALL ORDERS MUST GO THROUGH THE PRECHECK! The PreCheck page handles CSV files containing product information and personalization data. It allows users to upload a CSV file, preview and edit the personalization fields, validate SKUs, and download an updated CSV file with the changes.

  1. Locate and open the CSV file obtained from ShipStation
  2. Review each order meticulously, checking for any inconsistencies or anomalies.
  3. Take note of the different markings next to the order SKU, which indicate the following:
    • Green mark - The order is recognized and ready for designing.
    • Red mark - The order is unrecognized and cannot be designed.
    • Yellow mark - The order has a varying number of personalization lines, requiring close attention during the design process.
    • Red mark with a bug - The order has an unusual setup in the CSV file, which may lead to incorrect designing. Proceed with caution.
  4. Utilize the "Cancel" button to remove any items with a red mark from the CSV file. Please note that this action will only remove the individual item, not the entire order.

    5. By following these steps, you can efficiently manage and design orders based on their corresponding status indicators.

Designing and after-check with the Design page

The Designer page processes a CSV file containing product information and generates personalized images for each product based on the user's input. The images are saved to the user's 'Downloads' folder.

  1. Locate the "updated_csv" file in the Downloads folder, which was generated in the previous step, by pressing the browse button.
  2. Upload and submit the CSV file. Monitor the progress window at the bottom of the screen, which will confirm when the design process is complete.
  3. Access the newly created designs in the Downloads folder. These designs will be named with the current date and an index number.
  4. Thoroughly review each design to ensure its accuracy. This step serves as the final quality control checkpoint before proceeding to the printing stage. Exercise caution and attention to detail.
  5. In some departments, specific symbols are used during the shipping process to facilitate smoother operations. These symbols include:
    • Rabbit: This symbol indicates that the order is either expedited or a redo, requiring prompt attention and faster processing.
    • Shipping box: This symbol signifies that the order contains more than one item. If possible, these items should be kept together to maintain efficiency.
    • Multiple cubes: This symbol represents that the order consists of multiple instances of the same design on different items. It is crucial to ensure consistency across the various items.
    • Multiple tags: This symbol denotes that the order includes multiple SKUs. Pay close attention to the different product types and their respective design requirements when processing these orders.

    6. By following these steps, you can efficiently manage and verify the design process, ensuring that all designs are accurate before moving on to the printing phase.

When to use the TemplateMerger

The Template Merger Page is for processing and combining multiple images loaded by the user. It also provides a preview for the final combined image.When processing large batches of prints simultaneously, follow these steps using the TemplateMerger:

  1. Select the appropriate item format from the format bar, ensuring it matches the products to be printed on.
  2. Browse the Downloads folder to locate the accurate designs that were created and verified in the previous steps.
  3. Click "Export" to initiate the batch printing process. Please note that these files can be quite large, and it may take some time for the browser to generate them.
  4. If you encounter an issue where the downloaded file size is "0 bytes" and appears blank when viewed as a .png, it might be a browser-specific problem. To resolve this issue:
    • Close your current browser.
    • Open a different web browser (e.g., Google Chrome, Mozilla Firefox, Microsoft Edge, or Safari).
    • Repeat the previous steps for creating and exporting large batches of prints using the TemplateMerger in the new browser.

    By following this procedure, you can efficiently create large batches of prints while maintaining consistency and quality throughout the process.
